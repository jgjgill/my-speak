# My Speak Native App 디자인 가이드라인
*모바일 언어 스피킹 학습을 위한 네이티브 디자인 시스템*

> 이 문서는 네이티브 앱 특화 UI/UX 가이드라인입니다. 공통 디자인 원칙은 [메인 디자인 가이드라인](./design-guidelines.md)을 참조하세요.

## 📱 디자인 철학

언어 스피킹 학습이라는 **행동 변화를 이끄는 콘텐츠**에 최적화된 모바일 디자인 시스템입니다. 세계 최고 모바일 디자이너들의 관점을 바탕으로 **콘텐츠와 터치 인터랙션의 완벽한 통합**을 통해 학습 효과를 극대화합니다.

## 🎯 네이티브 앱 특화 UI/UX 원칙

### 1. 몰입형 풀스크린 경험 (Immersive Fullscreen Experience)
**모바일 화면 제약을 장점으로 전환**:
- **전체화면 집중**: 단일 학습 태스크에 완전 몰입 환경 제공
- **상하단 공간 최대화**: Safe Area를 활용한 헤더/푸터 최적화
- **세로형 우선 설계**: 한 손 조작 가능한 세로 레이아웃 우선
- **콘텐츠 중심 UI**: 버튼보다 콘텐츠가 화면의 70% 이상 차지

### 2. 직관적 터치 네비게이션 (Intuitive Touch Navigation)
**모바일 네이티브 제스처 활용**:
- **스와이프 기반 진행**: 좌우 스와이프로 단계 간 이동
- **탭 vs 프레스 구분**: 단순 터치(재생) vs 길게 누르기(반복)
- **제스처 피드백**: 햅틱 피드백으로 터치 성공/실패 구분
- **자연스러운 뒤로가기**: iOS 왼쪽 엣지 스와이프, Android 백 제스처 지원

### 3. 맥락적 네비게이션 (Contextual Navigation)
**모바일 앱만의 자연스러운 흐름**:
- **스택 기반 네비게이션**: 단선적 학습 진행을 위한 스택 내비게이션
- **모달 기반 보조 기능**: 설정, 도움말은 모달로 오버레이
- **탭 바 최소화**: 하단 탭 대신 진행 상태 기반 네비게이션
- **상태 유지**: 앱 전환 후 복귀 시 정확한 학습 위치 유지

## 🎯 My Speak 네이티브 앱 디자인 시스템

### React Native + Expo 아키텍처 활용
**하이브리드 웹뷰 중심 설계**:
- **네이티브 헤더**: 로고, 프로필, 뒤로가기 - 플랫폼 일관성 유지
- **웹뷰 메인 영역**: 학습 콘텐츠는 웹 기술로 풍부한 인터랙션 제공
- **네이티브 하단 영역**: 시스템 Safe Area 및 제스처 영역 보호
- **세션 브릿지**: Supabase 인증 정보 네이티브 ↔ 웹뷰 동기화

### 네이티브 헤더 디자인 (NativeHeader)
**플랫폼 공통 헤더 컴포넌트**:
```typescript
// 헤더 구조: 56pt 고정 높이
┌─[뒤로가기]─┬──────[My Speak 로고]──────┬─[프로필]─┐
│    44pt    │          flex: 2         │   44pt   │
│ chevron-   │       fontSize: 20       │ person-  │
│ back       │       fontWeight: bold   │ circle   │
└────────────┴──────────────────────────┴──────────┘
```

### 터치 영역 최적화 실제 적용
**iOS HIG 44pt 최소 터치 영역 준수**:
- **뒤로가기 버튼**: 44pt × 44pt, activeOpacity: 0.7
- **프로필 버튼**: 44pt × 44pt, 아바타 36pt 내부 배치
- **로고 영역**: 터치 불가, 시각적 브랜딩 용도
- **헤더 전체**: 56pt 높이, 하단 1px 구분선

### SafeAreaProvider 활용
**react-native-safe-area-context 기반 레이아웃**:
```typescript
<SafeAreaProvider>
  <SafeAreaView style={{ flex: 1, backgroundColor: "#FFFFFF" }}>
    <NativeHeader />        // 네이티브 네비게이션
    <SimpleWebView />       // 학습 콘텐츠 웹뷰
  </SafeAreaView>
</SafeAreaProvider>
```

### Expo Router 네비게이션 패턴
**Stack 기반 페이지 라우팅**:
- **/** (index): 메인 학습 화면 (헤더 + 웹뷰)
- **/login**: 로그인 전용 페이지
- **/profile**: 프로필 관리 페이지
- **headerShown: false**: 모든 화면에서 커스텀 헤더 사용

### 색상 시스템 (실제 적용값)
**My Speak 브랜드 컬러**:
```typescript
const colors = {
  primary: "#1E40AF",      // 로고, 뒤로가기 버튼
  secondary: "#3B82F6",    // 프로필 아이콘, 아바타 배경
  background: "#FFFFFF",   // 헤더, 메인 배경
  border: "#E5E7EB",      // 헤더 하단 구분선
  text: "#1E40AF",        // 로고 텍스트
}
```

## 📐 My Speak 레이아웃 시스템

### 하이브리드 앱 레이아웃 구조
```
┌─────────────────────────┐ ← Status Bar (Dynamic)
│    SafeAreaView Top     │ ← 시스템 영역
├─────────────────────────┤
│    NativeHeader (56pt)  │ ← 네이티브 헤더 고정
│ ┌─────┬─────────┬─────┐ │
│ │Back │My Speak │Prof │ │
│ └─────┴─────────┴─────┘ │
├─────────────────────────┤
│                         │
│    SimpleWebView        │ ← 학습 콘텐츠 웹뷰 (flex: 1)
│    (Web Content)        │   - 5단계 학습 시스템
│                         │   - 번역 입력 폼
│                         │   - 진행 상황 표시
│                         │
├─────────────────────────┤
│   SafeAreaView Bottom   │ ← Home Indicator 영역
└─────────────────────────┘
```

### 네이티브-웹뷰 경계 관리
**명확한 역할 분담**:
- **네이티브 영역**: 플랫폼 네비게이션, 시스템 통합, Safe Area
- **웹뷰 영역**: 학습 콘텐츠, 복잡한 UI 인터랙션, 반응형 레이아웃
- **데이터 브릿지**: 인증 정보, 진행 상황 동기화

### 인증 상태별 UI 패턴
**조건부 렌더링 시스템**:
```typescript
// 로그인 전: person-circle-outline 아이콘
// 로그인 후: 실제 Google 아바타 이미지 또는 person 아이콘

{user ? (
  <Image source={{ uri: user.user_metadata?.avatar_url }} />
) : (
  <Ionicons name="person-circle-outline" size={32} />
)}
```

### 로딩 상태 디자인
**ActivityIndicator 중심 로딩 UI**:
- **전면 로딩**: 인증 상태 확인 중 화면 중앙 스피너
- **부분 로딩**: 웹뷰 내부 콘텐츠 로딩은 웹 기술 활용
- **배경색 통일**: #FFFFFF로 네이티브-웹뷰 일관성 유지

## 🔄 터치 인터랙션 패턴

### 기본 제스처 정의
| 제스처 | 기능 | 적용 영역 | 피드백 |
|--------|------|-----------|---------|
| 탭 | 재생/선택 | 전체 콘텐츠 영역 | 햅틱 + 시각적 |
| 길게 누르기 | 반복 재생 | 오디오/텍스트 영역 | 진동 + 애니메이션 |
| 좌우 스와이프 | 이전/다음 단계 | 화면 가장자리 | 페이지 전환 |
| 위아래 스와이프 | 콘텐츠 스크롤 | 메인 영역 | 스크롤 인디케이터 |
| 핀치 | 텍스트 크기 조절 | 읽기 영역 | 실시간 크기 변경 |

### 상태 피드백 시스템
- **로딩 상태**: 스켈레톤 UI + 펄스 애니메이션
- **성공 상태**: 초록색 체크마크 + 성공 햅틱
- **오류 상태**: 빨간색 강조 + 오류 햅틱 + 안내 텍스트
- **진행 상태**: 프로그레스 바 + 단계별 완료 마커

### My Speak 특화 터치 인터랙션
**하이브리드 앱 제스처 디자인**:
- **네이티브 버튼**: 헤더 영역 44pt 터치 영역, activeOpacity: 0.7
- **웹뷰 콘텐츠**: 5단계 학습 시스템의 모든 터치 이벤트
- **경계 처리**: 네이티브-웹뷰 스크롤 영역 명확한 분리
- **키보드 대응**: 웹뷰 내 입력 시 자연스러운 키보드 표시

### 시각적 상태 피드백
**일관된 피드백 디자인**:
- **로딩 상태**: ActivityIndicator 중앙 배치
- **터치 피드백**: 모든 버튼 동일한 activeOpacity 0.7
- **인증 상태**: 프로필 영역 아바타/아이콘 조건부 표시
- **네비게이션**: iOS 스와이프백, Android 하드웨어 백 지원

```typescript
const MySpeak = {
```

### 타이포그래피 시스템
**네이티브 폰트 활용**:

- **헤더 텍스트**: 시스템 기본 폰트 (SF Pro/Roboto)
- **아이콘**: Ionicons 24pt-32pt 크기 일관성
- **웹뷰 내용**: 웹 타이포그래피 시스템 준수

### 아이콘 시스템
**Expo Vector Icons 활용**:
- **뒤로가기**: `chevron-back` 24pt, color primary
- **프로필 (로그인 전)**: `person-circle-outline` 32pt
- **프로필 (로그인 후)**: `person` 20pt 또는 아바타 이미지
- **일관성**: 모든 아이콘 동일한 컬러 팔레트 사용

## ⚠️ 웹 vs 네이티브 디자인 차별점

### 네이티브 앱에서 자연스러운 UI 패턴
- ✅ **네이티브 헤더** - 플랫폼별 표준 네비게이션
- ✅ **풀스크린 몰입** - 브라우저 UI 없는 순수 학습 환경
- ✅ **터치 최적화** - 44pt 최소 터치 영역, activeOpacity 피드백
- ✅ **시스템 통합** - Safe Area, 상태바, 키보드 자동 처리
- ✅ **플랫폼 제스처** - iOS 스와이프백, Android 하드웨어 백

### 네이티브 앱에서 부자연스러운 패턴 (웹 전용)
- ❌ **URL 주소창** - 앱에서는 의미 없는 브라우저 개념
- ❌ **브레드크럼** - 웹식 계층 네비게이션과 상충
- ❌ **마우스 호버** - 터치 인터랙션에서 구현 불가능
- ❌ **키보드 단축키** - 소프트웨어 키보드 환경에서 부자연스러움
- ❌ **우클릭 메뉴** - 터치 기반 롱프레스와 다른 개념

### 크로스 플랫폼 일관성 유지
- **색상 시스템**: 웹-네이티브 동일한 브랜드 컬러 사용
- **학습 플로우**: 5단계 시스템 구조 플랫폼 무관 동일
- **타이포그래피**: 웹뷰 내 폰트는 웹 가이드라인 준수
- **아이콘**: Ionicons로 플랫폼 간 시각적 통일성
