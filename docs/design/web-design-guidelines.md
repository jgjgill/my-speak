# My Speak 웹 애플리케이션 디자인 가이드라인
*대화면 학습 경험을 위한 웹 특화 UI/UX*

> 이 문서는 웹 플랫폼 특화 UI/UX 가이드라인입니다. 공통 디자인 원칙은 [메인 디자인 가이드라인](./design-guidelines.md)을 참조하세요.

## 💻 웹 플랫폼 UI/UX 특화 원칙

### 1. 시각적 풍부함 활용 (Visual Richness Utilization)
**웹의 대화면 강점 극대화**:
- **음파 시각화**: 마이크 입력 시 실시간 음성 파형 표시
- **복잡한 CSS 애니메이션**: transition, transform을 활용한 부드러운 학습 전환
- **멀티레이어 정보**: 메인 학습 + 사이드 정보 동시 표시
- **시각적 비교**: 번역 전후 비교를 위한 좌우 대칭 레이아웃

### 2. 키보드 중심 인터랙션 (Keyboard-Centric Interaction)
**웹 네이티브 경험 최적화**:
- **탭 네비게이션**: 논리적 포커스 순서로 모든 학습 요소 접근
- **타이핑 최적화**: 번역 입력을 위한 텍스트 필드 포커스 관리

### 3. 멀티태스킹 지원 (Multi-tasking Support)
**데스크톱 환경 활용**:
- **3단 레이아웃**: 사이드바(진도) + 메인(학습) + 사이드(도구)
- **플로팅 패널**: 사전, 발음 가이드 등 보조 도구 오버레이
- **브라우저 탭 활용**: 여러 학습 세션 동시 진행 지원
- **상황별 툴팁**: 마우스 호버 시 상세 정보 제공

## 🎯 My Speak 서비스 디자인 시스템

### 5단계 학습 시각적 차별화
**단계별 고유 컬러 시스템**:
- **1단계 (번역)**: 파란색 계열 - 입력과 사고를 위한 차분한 색상
- **2단계 (문장완성)**: 초록색 계열 - 성장과 발전을 나타내는 색상  
- **3단계 (스피킹)**: 주황색 계열 - 활동적이고 에너지 넘치는 색상
- **4단계 (자유발화)**: 보라색 계열 - 창의성과 자유로움을 표현
- **완료 상태**: 골드 계열 - 성취감과 완주의 의미

### 번역 입력 UI 디자인
**텍스트 입력 최적화 레이아웃**:
- **넓은 입력 영역**: 웹의 가로 공간 활용한 와이드 텍스트 필드
- **실시간 피드백**: 입력 중 정답률 색상 변화 (빨강→노랑→초록)
- **키보드 포커스**: 뚜렷한 포커스 링과 그림자 효과
- **플레이스홀더**: 회색 텍스트로 입력 가이드 제공
- **자동완성 드롭다운**: 입력 필드 하단 추천 번역 표시

### 한국어-언어 하이라이터 디자인
**이중 언어 텍스트 시각화**:
- **구문별 구분**: 배경색으로 번역 단위 시각적 분리
- **호버 효과**: 마우스 오버 시 연결된 영어 구문 하이라이트
- **색상 매핑**: 한국어-언어 구문 간 일치하는 색상 코드
- **행간 조정**: 두 언어 혼재 시 최적 가독성을 위한 line-height

### 진행 상황 시각화 디자인
**웹 특화 프로그레스 인디케이터**:
- **멀티레벨 프로그레스**: 전체 토픽 → 현재 단계 → 세부 포인트
- **원형 차트**: CSS conic-gradient 활용한 진도 원형 표시
- **단계별 아이콘**: 완료(체크), 진행중(점), 대기(원) 상태 표시
- **색상 그라데이션**: 진행률에 따른 색상 변화 효과
- **호버 상세정보**: 마우스 오버 시 상세 진행 정보 툴팁

## 📱 네이티브-웹뷰 디자인 차별화

### 웹뷰 환경 감지 UI 조정
**플랫폼별 헤더 디자인**:
- **브라우저 모드**: 로고, 네비게이션, 로그인 버튼 포함 풀 헤더
- **웹뷰 모드**: 헤더 숨김, 네이티브 앱 네비게이션에 의존
- **세이프 에어리어**: iOS 노치/홈 인디케이터 영역 고려한 패딩
- **스크롤 경계**: 웹뷰 내 바운스 스크롤 제어

### 반응형 버튼 시스템
**터치/마우스 환경 대응**:
```css
/* 데스크톱: 마우스 최적화 */
.btn-desktop { 
  min-height: 40px; 
  padding: 12px 24px; 
  hover:shadow-lg; 
}

/* 모바일/웹뷰: 터치 최적화 */
.btn-mobile { 
  min-height: 48px; 
  padding: 16px 32px; 
  active:scale-95; 
}
```

## 🌈 색상 시스템 & 브랜딩

### 학습 상태별 색상 팔레트
```css
:root {
  /* 단계별 주요 색상 */
  --stage-1: #3B82F6; /* 파랑 - 번역 */
  --stage-2: #10B981; /* 초록 - 문장완성 */
  --stage-3: #F59E0B; /* 주황 - 스피킹 */
  --stage-4: #8B5CF6; /* 보라 - 자유발화 */
  --completed: #F59E0B; /* 골드 - 완료 */
  
  /* 피드백 색상 */
  --correct: #10B981;
  --incorrect: #EF4444;
  --partial: #F59E0B;
  --neutral: #6B7280;
}
```

### 반응형 텍스트 크기
- **모바일**: `text-base` (16px) 기본, 입력 필드 16px 이상
- **태블릿**: `sm:text-lg` (18px) 학습 텍스트 가독성 향상
- **데스크톱**: `lg:text-xl` (20px) 대화면 활용한 큰 텍스트
- **사용자 조절**: 설정을 통한 폰트 크기 개별 조정 지원

## ⚠️ 플랫폼별 UX 차별점 & 베스트 프랙티스

### 웹에서 자연스러운 패턴
- ✅ **브레드크럼 네비게이션** - 토픽 > 단계 > 현재 위치 표시
- ✅ **다중 창/탭** - 여러 토픽 동시 학습 세션 지원
- ✅ **마우스 호버 상태** - 단어별 상세 정보 미리보기
- ✅ **키보드 단축키** - Space(재생), Enter(다음), ESC(취소)
- ✅ **URL 공유** - 특정 학습 단계 직접 링크 공유
- ✅ **텍스트 선택/복사** - 학습 내용 외부 활용 가능
- ✅ **우클릭 메뉴** - 문맥 기반 빠른 액션 제공
- ✅ **브라우저 뒤로가기** - 자연스러운 네비게이션 지원

### 웹에서 부자연스러운 패턴 (모바일 앱 전용)
- ❌ **스와이프 제스처** - 마우스 환경에서 직관적이지 않음
- ❌ **길게 누르기** - 웹에서는 우클릭이 더 자연스러움
- ❌ **풀스크린 강제** - 브라우저 UI와 충돌하여 사용자 혼란
- ❌ **하드웨어 백 버튼** - 브라우저 뒤로가기와 혼동 야기
- ❌ **터치 전용 제스처** - 핀치 줌, 회전 등 마우스로 불가능
- ❌ **진동 피드백** - 대부분 브라우저/데스크톱에서 미지원

### 크로스 플랫폼 일관성 유지
- **공통 색상 시스템**: 브랜드 색상과 단계별 색상 통일
- **동일한 학습 플로우**: 5단계 구조는 플랫폼 무관하게 동일
- **일관된 피드백**: 성공/실패 시각적 반응 통일
- **동기화된 진행 상황**: 플랫폼 간 학습 데이터 실시간 동기화
