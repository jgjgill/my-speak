# My Speak 디자인 가이드라인
*언어 스피킹 학습을 위한 콘텐츠 중심 디자인 시스템*

## 🎯 디자인 철학

언어 스피킹 학습이라는 **행동 변화를 이끄는 콘텐츠**에 최적화된 디자인 시스템입니다. 세계 최고 디자이너들의 관점을 바탕으로 **콘텐츠와 디자인 요소의 본질적 연결**을 통해 학습 효과를 극대화합니다.

## 🧠 콘텐츠-디자인 연결 원칙

### 1. 음성 학습의 다감각 번역 (Audio Learning Multi-Sensory Translation)
**콘텐츠 특성**: 실시간 음성 인터랙션, 청각적 학습
**디자인 연결**:
- **실시간 피드백**: 마이크 입력 시 즉각적인 시각적/햅틱 피드백
- **리듬감 있는 UI**: 음성의 리듬과 강세를 반영한 시각적 패턴과 애니메이션
- **정확도 시각화**: 발음 정확도를 색상 그라데이션으로 실시간 표현
- **대화적 인터페이스**: 실제 대화처럼 자연스러운 흐름이 있는 레이아웃

### 2. 인지 부하 최적화 (Cognitive Load Optimization)
**콘텐츠 특성**: 동시 처리되는 한국어→대상언어 번역, 발음, 문법 학습
**디자인 연결**:
- **정보 계층의 시간적 배치**: 한국어 문장 → 번역 시간 → 대상언어 표시 순서적 노출
- **인지적 청킹**: 긴 문장을 의미 단위로 시각적 분할하여 기억 부담 감소
- **듀얼 코딩**: 텍스트와 시각적 단서(색상, 아이콘)의 조합으로 기억 강화
- **주의 집중 가이드**: 현재 학습 요소만 강조하고 나머지는 시각적으로 억제

### 3. 학습 동기의 시각적 구현 (Motivation Visualization)
**콘텐츠 특성**: 5단계 순차 학습, 성취감, 지속적 동기 부여 필요
**디자인 연결**:
- **진행의 게임화**: RPG 게임의 경험치 시스템을 모방한 단계별 진행 표시
- **성취의 의식화**: 단계 완료 시 축하 애니메이션과 성취 배지로 도파민 자극
- **사회적 압박 활용**: 학습 연속일수, 목표 달성률 등 시각적 지표로 지속 동기 제공
- **개인화된 격려**: 사용자 학습 패턴을 반영한 맞춤형 격려 메시지와 시각적 피드백

### 4. 언어 학습의 문화적 맥락 (Cultural Context in Language Learning)
**콘텐츠 특성**: 한국어→대상언어 번역, 문화적 뉘앙스, 언어별 사고방식
**디자인 연결**:
- **언어별 색상 심리학**: 언어별 문화의 색상 선호도 반영 (파랑=신뢰, 빨강=에너지)
- **텍스트 방향성**: 한국어(좌→우)에서 대상언어(좌→우)로의 자연스러운 시각적 흐름
- **문화적 아이콘**: 언어별 문화를 상징하는 미묘한 시각적 요소 (글꼴, 패턴, 간격)
- **사회적 학습**: 대상언어 문화의 토론식 학습을 반영한 대화형 UI 패턴

## 🎨 세계 최고 디자이너 관점 적용

### Dieter Rams의 "Good Design" 10원칙 적용
**언어 스피킹 학습 맥락에서의 해석**:

1. **혁신적**: 기존 단어장 암기가 아닌 실제 대화 상황 시뮬레이션
2. **유용성**: 즉시 활용 가능한 실생활 표현 중심
3. **아름다움**: 학습 스트레스를 줄이는 시각적 편안함
4. **이해하기 쉬움**: 복잡한 문법 설명 없는 직관적 학습 흐름
5. **비간섭적**: 학습 콘텐츠를 방해하지 않는 미니멀 UI
6. **정직함**: 실제 발음 정확도를 과장 없이 정확하게 피드백
7. **지속성**: 유행을 타지 않는 클래식한 학습 인터페이스
8. **디테일**: 발음 기호, 강세 표시 등 미세한 학습 도구까지 고려
9. **환경 친화적**: 종이 교재 대신 디지털로 지속 가능한 학습
10. **최소한**: 학습에 꼭 필요한 요소만 남기고 모든 장식 제거

### Don Norman의 감정적 디자인 3단계
**언어 스피킹 학습에서의 감정적 여정**:

1. **본능적 수준**: 발음할 때의 즉각적 만족감을 위한 실시간 피드백
2. **행동적 수준**: 반복 학습이 지루하지 않도록 하는 변화 있는 인터랙션
3. **성찰적 수준**: 언어 실력 향상에 대한 자부심과 정체성 강화

### Steve Jobs의 "Simplicity is Sophistication"
**복잡한 언어 학습을 단순하게**:
- 5단계 학습을 하나의 직선적 여정으로 시각화
- 복잡한 문법 규칙을 색상과 패턴으로 단순화
- 다단계 메뉴 대신 직관적인 인터랙션으로 모든 기능 접근

### Jan Tschichold의 타이포그래피 원칙
**언어 학습에 특화된 문자 디자인**:
- **대비**: 한국어(진한 색상)와 학습 언어(연한 색상)의 명확한 구분
- **정렬**: 번역 과정의 시각적 흐름을 나타내는 정렬 시스템
- **근접성**: 관련된 단어들을 그룹핑하여 기억 패턴 강화
- **반복**: 일관된 타이포그래피로 학습 패턴 형성

## 💡 핵심 디자인 원칙

### 1. 몰입 우선 설계 (Immersion-First Design)
- **학습 플로우 보호**: 외부 방해 요소 완전 차단
- **단일 작업 집중**: 한 번에 하나의 학습 단계만 노출
- **시각적 노이즈 제거**: 장식적 요소 배제, 기능적 미니멀리즘

### 2. 인지 과학 기반 정보 설계 (Cognitive Science-Based Information Design)
- **작업 기억 고려**: 7±2 법칙에 따른 정보 청킹
- **이중 부호화 이론**: 텍스트와 시각 정보의 최적 조합
- **인지 부하 이론**: 본질적/외재적/생성적 인지 부하 균형

### 3. 행동 변화 유도 설계 (Behavior Change Design)
- **습관 형성 루프**: 신호→루틴→보상 사이클 구현
- **점진적 공개**: 사용자 역량에 따른 기능 단계적 해제
- **사회적 증명**: 학습 진도와 성취 공유 시스템

## 🎨 색상 시스템: 학습 심리학 기반

### 학습 상태별 색상 코드
- **집중 상태**: `#1E40AF` (깊은 파랑 - 집중력 향상, 신뢰감)
- **활동 상태**: `#3B82F6` (밝은 파랑 - 에너지, 상호작용)
- **성취 상태**: `#10B981` (초록 - 성공, 완료, 긍정적 피드백)
- **주의 상태**: `#F59E0B` (주황 - 경고, 집중 필요)
- **오류 상태**: `#EF4444` (빨강 - 오류, 재시도 필요)

### 언어별 문화 색상
- **한국어 영역**: `#1F2937` (차분한 회색 - 모국어의 안정감)
- **학습 언어 영역**: `#1E40AF` (신뢰의 파랑 - 새로운 언어 도전)
- **번역 과정**: `#6B7280` (중간 회색 - 사고 과정 표현)

### 인지 부하별 색상 강도
- **높은 집중 필요**: 선명한 대비 (90% 채도)
- **보조 정보**: 중간 대비 (60% 채도)  
- **배경 요소**: 낮은 대비 (30% 채도)

## ✍️ 타이포그래피

### 글꼴 스택
```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', 
             'Noto Sans KR', sans-serif;
```

### 텍스트 크기 시스템
- **Hero (36px+)**: 메인 타이틀, 로고
- **Title (24-32px)**: 페이지 제목, 섹션 헤더
- **Heading (18-20px)**: 카드 제목, 중요 정보
- **Body (16px)**: 기본 본문 텍스트
- **Caption (14px)**: 보조 정보, 메타데이터
- **Small (12px)**: 라벨, 안내문구

### 텍스트 스타일
- **한국어**: 자간 `-0.02em`, 행간 `1.6`
- **대상언어**: 자간 `normal`, 행간 `1.5`
- **코드/API**: `font-mono`, 배경색으로 구분

## 🎭 학습 경험 특화 가이드라인

### 1. 문장 표시
- **한국어 문장**: 18px, 진한 텍스트 (#1F2937)
- **대상언어 문장**: 16px, 보조 텍스트 (#6B7280)
- **강조 단어**: 볼드 처리 + 브랜드 컬러

### 2. 진행도 표시
- **단계별 진행**: 원형 인디케이터 + 연결선
- **완료 표시**: 체크마크 + 초록색 배경
- **현재 단계**: 브랜드 컬러 + 애니메이션

### 3. 피드백 시스템
- **정답**: 초록색 배경 + 체크 아이콘
- **오답**: 빨간색 배경 + X 아이콘
- **힌트**: 노란색 배경 + 전구 아이콘

## ⚡ 애니메이션 가이드라인

### 전환 효과
- **기본 duration**: `200ms`
- **복잡한 전환**: `300ms`
- **Easing**: `ease-out` (일반), `ease-in-out` (모달)

### 마이크로 인터랙션
- **성공 피드백**: 부드러운 확대 + 색상 변화
- **오류 처리**: 좌우 흔들림 애니메이션
- **로딩**: 부드러운 페이드 인/아웃

## 🔧 구현 가이드라인

### 상태 관리
- **로딩 상태**: 스켈레톤 UI 제공
- **에러 상태**: 친화적인 에러 메시지
- **빈 상태**: 액션 가능한 빈 상태 화면

### 접근성
- **색상 대비**: 최소 4.5:1 대비율 유지
- **키보드 네비게이션**: 모든 인터랙티브 요소 접근 가능
- **스크린 리더**: 적절한 시맨틱 마크업과 ARIA 레이블
